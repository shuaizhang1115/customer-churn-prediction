# Customer-churn-prediction
End-to-end customer churn prediction project using Logistic Regression and Random Forest

## Project Overview
This project predicts customer churn using machine learning and translates the results into business recommendations.

## Objectives
- Predict which customers are likely to churn
- Focus on recall because missing churn customers is costly
- Improve model performance through threshold tuning
- Identify key churn drivers

## Dataset
- Telco Customer Churn Dataset from Kaggle
- Around 7,000 rows
- Target variable: Churn
- Imbalanced classification problem

## Workflow
1. Data cleaning
2. Encoding
3. Feature engineering
4. Logistic Regression
5. Random Forest
6. ROC / AUC
7. Threshold tuning
8. Feature importance
9. Business insights

## Key Findings
- Month-to-month customers are more likely to churn
- High monthly charges are associated with higher churn
- Fiber optic users show higher churn risk
- Electronic check users churn more often
- Low-tenure customers are high-risk

## Business Recommendations
- Target high-risk customers early
- Encourage longer-term contracts
- Improve fiber service experience
- Improve payment experience
- Strengthen onboarding for new customers
